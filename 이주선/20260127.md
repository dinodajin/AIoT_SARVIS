# SARVIS Frontend – 개념 정리 + 삽질 로그 통합 README


이 문서는 SARVIS 프론트엔드 개발을 진행하면서  
**React / React Native / Expo / CLI 개념을 정리하고**,  
직접 환경을 구축하며 겪었던 **삽질 경험을 기록**한 문서이다.

"왜 안 되는지 몰라서 헤맸던 시간"을 줄이기 위해 작성함.

---

## 1. 내가 실제로 한 것 (환경 구성)

SARVIS 프론트엔드는 **React Native + Expo** 기반으로 구성했다.

### 설치 및 실행 과정
- Node.js 설치
- Expo CLI 사용
- React Native 프로젝트 생성
- Android Emulator 및 실제 안드로이드 기기 연결

```bash
npm install
npx expo start
npx expo start --tunnel
```

👉 이 과정을 통해:
- Android Studio 에뮬레이터 실행
- 실제 안드로이드 폰(Expo Go) 실행
- 웹(Web) 환경 실행  
을 모두 경험했다.

## 2. React / React Native / Expo / CLI 차이 정리

### React
> **웹 UI를 만들기 위한 라이브러리**

- HTML 태그 사용 (`div`, `span`)
- 브라우저에서 실행
- 웹 서비스 개발용

---

### React Native
> **React 문법으로 모바일 앱 UI를 만드는 기술**

- Android / iOS 앱 개발
- `View`, `Text`, `Image` 사용
- 웹 코드 그대로 사용 불가

👉 **역할**
- 화면(UI) 구성
- 상태 관리
- 사용자 입력 처리

---

### Expo
> **React Native를 쉽게 쓰게 해주는 도구 세트**

- 복잡한 네이티브 설정을 대신 처리
- QR 코드로 바로 실행 가능
- 에뮬레이터 / 실제 폰 테스트 지원

👉 React Native 위에서 동작함

---

### CLI (Command Line Interface)
> **터미널 명령어로 프로젝트를 조작하는 방식**

- 프로젝트 생성
- 실행
- 빌드

```bash
npx expo start
```

👉 Expo를 제어하기 위한 수단일 뿐, 기술 자체는 아님

## 3. React Native vs Kotlin (역할 이해)

### React Native
- **UI / 화면 담당**
- 하나의 코드로 Android / iOS 대응
- 빠른 개발, 화면 중심 로직에 적합

### Kotlin
- **안드로이드 네이티브 로직 담당**
- 하드웨어 제어, 시스템 깊은 접근
- 카메라, 블루투스, 센서, 성능 최적화

👉 현재 프로젝트에서는:
- **React Native**: 화면과 사용자 인터랙션
- **Kotlin**: 필요 시 네이티브 기능 확장용

> 즉, RN이 "앞단", Kotlin이 "안쪽 엔진" 역할

## 4. 웹은 되는데 안드로이드는 안 되던 문제

### 증상
- 웹에서는 화면 정상
- 안드로이드에서는 기본 화면만 출력
- 코드 수정해도 변화 없음

### 원인
- Expo Router 구조 이해 부족
- 안드로이드는 `app/(tabs)/index.tsx`만 렌더링

### 깨달은 점
> 안드로이드는 App.tsx가 아니라  
> app/ 폴더 안의 route 파일을 기준으로 화면을 그린다.

---

## 5. 코드가 안 되는 줄 알았는데 사실은 안 보고 있었던 문제

- 로직 문제라고 착각
- 실제 원인은 **파일 위치**
- 안드로이드가 보지 않는 파일을 수정 중이었음

👉 구조부터 확인하는 습관의 중요성 인지

---

## 6. `@/` alias 때문에 조용히 망가졌던 문제

### 증상
- 웹 OK
- 안드로이드 화면 안 뜸
- 에러 메시지 없음

### 원인
```ts
import { Something } from '@/components/...';
```

- Metro 번들러에서 alias가 조용히 실패

### 해결
- 상대 경로 사용

```ts
import { Something } from '../../components/...';
```

---

## 7. 에뮬레이터는 되는데 실제 폰은 안 되던 문제

### 원인
- LTE / 핫스팟 환경
- Expo 기본 실행은 LAN 모드

### 해결
```bash
npx expo start --tunnel
```

- Tunnel 모드에서만 실제 폰 접속 가능
- `exp.direct` 주소 확인

---

## 8. 최종적으로 얻은 교훈

- 웹에서 되는 코드 ≠ 모바일에서 되는 코드
- 화면 안 바뀌면 코드보다 **구조부터 확인**
- Expo는 에뮬레이터보다 실제 기기 테스트가 중요
- React Native와 Kotlin의 역할은 명확히 다름
